# OpenTofu AWS Scaffold

The future pilot is planned for AWS `eu-north-1` and will use separate accounts or equivalent
strong boundaries for development, staging, and production.

This directory currently creates no resources. Before implementation it requires:

- Remote state and locking design.
- Account and organization structure.
- Network and private endpoint design.
- OIDC roles and permissions.
- RDS backup and restore tests.
- KMS signing and rotation procedures.
- Cost budgets and alerts.
- Legal approval of all subprocessors and data flows.
